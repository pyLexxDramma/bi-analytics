# Журнал правок по скриншотам и ТЗ (апрель 2026)

Сопоставление с папкой `Правки/screens/file (1)/Правки (1)` (file-001…024): макеты предписаний и девелоперских проектов, маппинг БДДС/вех, ГДРС/СКУД, уточнения по отклонениям и контрольным точкам.

## Выполнено в этом коммите

1. **П.2 — «Выбор панели»**  
   Убран блок «### Выбор панели» и радио-группы в основной области. Текущий отчёт задаётся только боковым меню; при несовпадении с доступными отчётами выбирается первый доступный. Флаг `dashboard_selected_from_menu` больше не используется при клике по меню.

2. **П.3 — фон графиков**  
   В `apply_chart_background` для `plot_bgcolor` / `paper_bgcolor` задан цвет `hsl(209,67%,22%)` (`CHART_BG_COLOR`), согласованный с основным фоном панели в `static/css/style.css`. Таблицы по-прежнему используют чуть более тёмный `TABLE_BG_COLOR` для читаемости.

3. **П.1 / П.4 — ГДРС (частично)**  
   - План/факт по проекту: вместо круговой диаграммы, скрывающей нулевой «Факт», выводится **столбчатая** диаграмма «План / Факт» с подписями над столбцами.  
   - Добавлено распознавание колонки плана по шаблону **«Количество… работник / спецтех / техник / ресурс»** в `_gdrs_resolve_plan_column`.  
   - Скрыта круговая диаграмма «Распределение суммы Плана и Среднего за месяц по контрагентам» (в двух местах кода) — остаётся сводная таблица.  
   - Вкладка **«Учёт по сменам»** переименована в **«СКУД по неделям»** (соответствие блоку «Факт СКУД» по неделям в макете; «смены» в названии не используются).  
   - **«Динамика людей и техники»:** исправлена фильтрация периодов — сравнение по `pd.Period` через словарь подписей `format_period_ru`, а не через `pd.Period(строка из selectbox)`.  
   - Колонка **«Период»:** в `format_period_ru` добавлена обработка `pd.NA`/`NaT` и строки `nan`.  
   - **Среднее за месяц** в сводной таблице динамики — целые числа, без двух знаков после запятой.

## Что проверить вручную на http://localhost:8501

- После загрузки данных отчёт открывается без блока «Выбор панели»; переключение только через меню слева.  
- ГДРС: столбцы «План / Факт», подписи; при нулевом факте столбец виден.  
- Вкладка «СКУД по неделям» и предупреждения при отсутствии файла ресурсов.  
- Динамика: в таблице нет `nan` в «Период», числа целые.  
- Визуальное совпадение фона графиков с фоном основной области.

## Шаг 2 (15.04.2026) — БДДС по лотам + даты в «Девелоперские»

- **БДДС «По периодам»:** для групповых столбцов заданы `bargap` / `bargroupgap`, чтобы реже наезжали подписи и ряды были плотнее.  
- **БДДС «По лотам»:** уменьшен лишний «воздух» по оси X (`~1.18` вместо `1.31`, `rangemode=tozero`), снижены правое поле и высота строки; добавлены `bargap` / `bargroupgap` для горизонтальных баров.  
- **Девелоперские проекты (`_fmt_date_ru`):** строки, похожие на чистое число, и годы 1900–2100 без контекста не выводятся как дата; даты со слэшем нормализуются к `дд.мм.гггг`. Строка «Выборка ДС» по-прежнему в млн руб. (не календарные даты).
