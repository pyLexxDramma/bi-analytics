# Benchmark двух локальных LLM-моделей

> Дата подготовки: 2026-04-09
> Статус: **методология + промпты готовы; таблицы результатов — заполнить после прогона**

---

## 1. Цель

Выбрать одну LLM для локального (on-premise / VPS) контура BI Analytics:
подсказки по проектным данным, генерация пояснений к отклонениям,
суммаризация таблиц, ответы на вопросы пользователей.

Сравниваем **два размера** семейства Qwen — «тяжёлый» (8 B) и «лёгкий» (4 B).

---

## 2. Модели

| # | Чекпойнт HuggingFace | Параметры | Контекст | Квантизация | Примечание |
|---|----------------------|-----------|----------|-------------|------------|
| A | `Qwen/Qwen3-8B` | 8.2 B | 32 768 (до 131 072 с YaRN) | bf16 / AWQ-4bit (`Qwen/Qwen3-8B-AWQ`) | основной кандидат «качество» |
| B | `Qwen/Qwen3-4B` | 4.0 B | 32 768 | bf16 / AWQ-4bit | кандидат «скорость + малый VRAM» |

> Если на сервере стоит Qwen **2.5**, замените чекпойнты:
> `Qwen/Qwen2.5-7B-Instruct` (7.6 B) и `Qwen/Qwen2.5-3B-Instruct` (3.1 B).
> Методология и промпты — те же.

---

## 3. Критерии оценки

| Критерий | Метрика | Вес | Как измеряем |
|----------|---------|-----|-------------|
| **Скорость генерации** | tokens/sec (output throughput, median по 3 прогонам) | 20 % | `benchmark_serving.py` или ручной замер |
| **Время до первого токена** | TTFT p50, мс | 10 % | `benchmark_serving.py --save-result` |
| **Пиковое потребление VRAM** | ГБ (nvidia-smi peak) | 15 % | `nvidia-smi --query-gpu=memory.used --format=csv -l 1` |
| **Общее качество** | средний балл 1–5 по 5 общим промптам | 15 % | ручная оценка (рубрика ниже) |
| **Доменное качество** | средний балл 1–5 по 15 доменным промптам | 30 % | ручная оценка (рубрика ниже) |
| **Следование формату** | % промптов, где ответ соответствует запрошенному формату (JSON / Markdown-таблица) | 10 % | автоматическая проверка + ручная |

**Итоговый балл** = взвешенная сумма нормализованных оценок (0–100).

## 4. Методология

### 4.1 Окружение (зафиксировать перед прогоном)

| Параметр | Значение |
|----------|---------|
| GPU | _(заполнить: модель, VRAM)_ |
| Драйвер NVIDIA | _(версия)_ |
| CUDA | _(версия)_ |
| vLLM | _(версия, напр. 0.8.x)_ |
| Движок | vLLM OpenAI-compatible API |
| `--dtype` | `auto` (bf16 на Ampere+; float16 на старых) |
| `--max-model-len` | 8192 |
| `--gpu-memory-utilization` | 0.90 |
| Температура | 0.3 (детерминизм) |
| `top_p` | 0.9 |
| `max_tokens` | 512 (для всех промптов единообразно) |

### 4.2 Прогоны

- Каждый промпт прогоняется **3 раза** для каждой модели.
- Между прогонами — пауза 5 с (остывание KV-cache).
- В отчёт идёт **медиана** по latency/tok-s и **лучший** из трёх ответов по качеству (консервативная оценка).

### 4.3 Оценка качества — рубрика

| Балл | Расшифровка |
|------|-------------|
| 5 | Ответ полностью верный, конкретный, соответствует формату |
| 4 | Верный по сути, мелкие стилистические недочёты или избыточность |
| 3 | Частично верный (>50 % фактов/тезисов корректны), формат соблюдён |
| 2 | Существенные ошибки или пропуски, но не полностью неверный |
| 1 | Галлюцинации, неверные факты, формат нарушен |

Оценку выставляет **один эксперт** (для устранения межэкспертной вариации).

## 6. Скрипты запуска

### 6.1 Запуск vLLM-сервера

```bash
# Модель A — Qwen3-8B (bf16)
python -m vllm.entrypoints.openai.api_server \
  --model Qwen/Qwen3-8B \
  --dtype auto \
  --max-model-len 8192 \
  --gpu-memory-utilization 0.90 \
  --port 8000

# Модель B — Qwen3-4B (bf16)
python -m vllm.entrypoints.openai.api_server \
  --model Qwen/Qwen3-4B \
  --dtype auto \
  --max-model-len 8192 \
  --gpu-memory-utilization 0.90 \
  --port 8000
```

> **AWQ-квантизация** (если VRAM < 16 ГБ):
> `--model Qwen/Qwen3-8B-AWQ --quantization awq`

### 6.2 Замер TTFT и tok/s (автоматический)

```bash
# Из клона https://github.com/vllm-project/vllm
python benchmarks/benchmark_serving.py \
  --model Qwen/Qwen3-8B \
  --tokenizer Qwen/Qwen3-8B \
  --endpoint /v1/chat/completions \
  --dataset-name sharegpt \
  --num-prompts 50 \
  --request-rate 1.0 \
  --save-result \
  --result-filename results_qwen3_8b.json
```

Результат содержит `mean_ttft_ms`, `median_ttft_ms`, `output_throughput` (tok/s).

### 6.3 Замер VRAM

```bash
# В параллельном терминале во время прогона
nvidia-smi --query-gpu=timestamp,memory.used,memory.total,utilization.gpu \
  --format=csv -l 1 | tee vram_log.csv
```

Пиковое значение `memory.used` — в отчёт.

### 6.4 Ручной прогон доменных промптов

```bash
# Простой curl-запрос для одного промпта (подставить текст)
time curl -s http://localhost:8000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "Qwen/Qwen3-8B",
    "messages": [
      {"role": "system", "content": "Ты — помощник аналитика. Отвечай на русском, кратко и по делу."},
      {"role": "user", "content": "ТЕКСТ_ПРОМПТА"}
    ],
    "temperature": 0.3,
    "top_p": 0.9,
    "max_tokens": 512
  }' | python -m json.tool
```

Прогоняет набор промптов через OpenAI-совместимый API (vLLM),

RUNS = 3

def run_bench(client, model: str, out_dir: pathlib.Path):
    results = []
    for prompt_id, prompt_text in PROMPTS.items():
        for run in range(1, RUNS + 1):
            t0 = time.perf_counter()
            resp = client.chat.completions.create(
                model=model,
                messages=[
                    {"role": "system", "content": SYSTEM},
                    {"role": "user", "content": prompt_text},
                ],
                temperature=0.3,
                top_p=0.9,
                max_tokens=512,
            )
            elapsed = time.perf_counter() - t0
            content = resp.choices[0].message.content or ""
            tok_out = resp.usage.completion_tokens if resp.usage else len(content.split())
            tok_in = resp.usage.prompt_tokens if resp.usage else 0
            results.append({
                "prompt_id": prompt_id,
                "run": run,
                "model": model,
                "input_tokens": tok_in,
                "output_tokens": tok_out,
                "elapsed_s": round(elapsed, 3),
                "tok_per_sec": round(tok_out / elapsed, 1) if elapsed > 0 else 0,
                "answer": content,
                "quality_score": None,  # заполнить вручную 1–5
            })
            print(f"  {prompt_id} run={run}  {tok_out} tok  {elapsed:.2f}s  {tok_out/elapsed:.1f} tok/s")
            time.sleep(5)

    ts = datetime.now().strftime("%Y%m%d_%H%M%S")
    safe_model = model.replace("/", "_")
    out_file = out_dir / f"bench_{safe_model}_{ts}.json"
    out_file.write_text(json.dumps(results, ensure_ascii=False, indent=2), encoding="utf-8")
    print(f"\nРезультаты → {out_file}")
    return results

def main():
    ap = argparse.ArgumentParser(description="LLM domain benchmark for BI Analytics")
    ap.add_argument("--model", required=True, help="Model name as registered in vLLM")
    ap.add_argument("--base-url", default="http://localhost:8000/v1", help="vLLM base URL")
    ap.add_argument("--out-dir", default="docs/bench_results", help="Output directory")
    args = ap.parse_args()

    out_dir = pathlib.Path(args.out_dir)
    out_dir.mkdir(parents=True, exist_ok=True)

    client = OpenAI(base_url=args.base_url, api_key="not-needed")
    print(f"=== Benchmark: {args.model} ({RUNS} runs x {len(PROMPTS)} prompts) ===\n")
    run_bench(client, args.model, out_dir)

if __name__ == "__main__":
    main()
