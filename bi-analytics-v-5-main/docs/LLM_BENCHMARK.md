# Сравнение локальных LLM для аналитики (шаблон отчёта)

Цель: выбрать модель для подсказок по проектам / текстам на стороне сервера (например, vLLM + Qwen).

## Модели к сравнению

| Модель | Размер | Контекст | Примечание |
|--------|--------|----------|------------|
| Qwen 2.5 (пример) | 8B | по GPU | основной кандидат «качество/скорость» |
| Qwen 2.5 (пример) | 3B | по GPU | легче по VRAM, быстрее |

(Точные имена чекпойнтов подставьте из своего реестра: `Qwen/Qwen2.5-...`.)

## Критерии

1. **Скорость:** измерять **time to first token (TTFT)** и **tokens/sec** при фиксированных `max_tokens`, батче 1, без очереди.
2. **Память:** **пиковая VRAM** (nvidia-smi) под нагрузкой; работа при заданном лимите (`gpu_memory_utilization` в vLLM).
3. **Качество (общее):** короткий набор промптов без домена (следование инструкции, JSON, избегание галлюцинаций).
4. **Качество (домен):** 10–20 вопросов по вашей аналитике: термины РД/ПД, отклонения, бюджет, таблицы — **ручная оценка** 1–5 или бинарно «факт верен / неверен».

## Методология

1. Один и тот же **движок** (vLLM), одна и та же **квантизация** (или обе bf16), одинаковые **температура / top_p**.
2. Для каждой модели: **3 прогона** на один и тот же промпт, в отчёт — медиана по latency и по tokens/sec.
3. Доменные ответы проверяет **один и тот же эксперт** по рубрике (таблица ниже).
4. Окружение зафиксировать: GPU, драйвер, версия CUDA, версия vLLM, длина контекста, длина входного текста (символы / токены).

## Итоговая рекомендация

- **Берём для прода:** _заполнить_ — указать модель и почему (баланс памяти, скорости и качества на доменных задачах).
- **Отклоняем:** _заполнить_ — где порог не пройден (медленно, мало памяти не хватает, сильные ошибки в цифрах).

## Команды-подсказки (vLLM)

```bash
# Пример запуска (подставьте model и порты)
python -m vllm.entrypoints.openai.api_server \
  --model Qwen/Qwen2.5-7B-Instruct \
  --dtype auto \
  --max-model-len 8192
```

Нагрузочный замер: вызовы к `/v1/chat/completions` с `stream=True` для TTFT и без стрима для среднего tok/s (по заголовкам/логам vLLM при необходимости).
