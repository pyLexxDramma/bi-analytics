## 1. Структура страницы

Один экран, вертикальная прокрутка, четыре блока:

1. **Фильтры** (горизонтальная панель + чекбокс):
   - Проект (Все проекты)
   - Подрядчик (Все подрядчики)
   - № договора (частичный поиск, текстовое поле)
   - **Период выданных предписаний** *(новое)* — диапазон дат
   - **Блок** *(новое)* — Блок А / Блок Б / Блок В (см. Q-27)
   - Чек-бокс **«Не отображать устранённые предписания»** *(новое)* —
     при включении в визуале и таблице остаются только неустранённые.

2. **Бар-чарт «Неустранённые предписания» по подрядчикам**
   (название блока поменять с «Предписания по подрядчикам» →
   «Неустранённые предписания»):
   - Горизонтальные оранжевые столбцы.
   - **Внутри столбца** — число просроченных по подрядчику.
   - **Синие пузырьки справа** — общее число неустранённых по подрядчику.
   - Легенда: «Внутри столбца — просроченные», «Синие цифры — все
     неустранённые».
   - Бар-чарт должен быть БОЛЬШЕ — все подписи и легенда видны без
     прокрутки/обрезки. При зуме страницы — горизонтальная скрол-полоса,
     а НЕ резайз диаграммы.

3. **Карточки «Ключевые показатели»**:
   - Неустранённые предписания (общее количество)
   - Просроченные предписания (требуют немедленного внимания)
   - Критические предписания (просрочка более 30 дней)
   - **Всего предписаний** *(новое)*
   - **Устранённые предписания** *(новое)*

4. **Таблица «Неустранённые предписания»** (детальная, под чартом):
   - Группировка по подрядчику с подписью «всего: N | крит: N».
   - Колонки:
     1. Подрядчик
     2. Проект
     3. № договора
     4. № предписания
     5. **Дата выдачи предписания** *(новое)*
     6. Срок устранения предписания
     7. **Фактическая дата устранения** *(новое)*
     8. Дней просрочки (отрицательные числа в красном фоне = просрочено)
     9. Критические предписания (бейдж «Критическое» / «—»)
     10. **Статус предписания** *(новое)*
   - **Сортировка**: подрядчик ↑, по просрочке ↓, критические — вверху.
   - **Раскраска строк**:
     - просрочено → фон строки розовый
     - устранено → фон строки салатовый

## 2. Источник данных

### Из `tessa_*-id.csv`
- Берётся `DocID` где `KindName = "Предписания"`.
- Исключаются документы с `KrStateID = 0` («Проект» — загружены, но
  процесс не запущен).
- Устранённые: `KrStateID = 13` («Снято»).

**Маппинг KrStateID → KrState** (проверено на `tessa_06-05-2026-00-07-id.csv`):
`0=Проект`, `1=На согласовании`, `2=Согласован`, `4=На доработке`,
`8=Подписан`, `9=Отказ`, `10=На подписании`, `12=На ознакомлении`,
`13=Снято`.

### Сопоставление с задачами `tessa_*-task.csv`
- Ключ: `CardID` в task-файле = `DocID` в id-файле.
- Условие: `TypeCaption = "Проверка"`, `OptionCaption = "Принято"`.
- `Completed` (из task) = **Фактическая дата устранения**.

### Срок устранения предписания
- В текущей выгрузке поля **нет** (заказчик подтвердил, добавит позже —
  «Как сделаем — отпишемся»). До этого момента колонка отображается
  как пустая, плашка «—».

### Дедупликация
- 1 `DocID` = 1 строка. Берём последний snapshot (по `Import_data` /
  `Import_date`). Логика та же, что в B-14 ИД (`_latest_snapshot`).

## 5. C-задачи (последовательность реализации)

1. **C-1** dedup по DocID + исключение `KrStateID = 4`. (P0-1, P0-2, P0-6)
2. **C-2** Подтянуть `№ договора` из `1C_ID_DOG` / `Lot` / `ContractNumber`. (P0-4)
3. **C-3** Подтянуть `Фактическая дата устранения` из task-файла:
   `CardID == DocID` ∧ `TypeCaption == "Проверка"` ∧
   `OptionCaption == "Принято"` → `Completed`.
4. **C-4** Добавить «Срок устранения предписания» — пока заглушка «—»,
   готовность подключить, как только заказчик добавит поле в выгрузку.
5. **C-5** Пересчитать «Дней просрочки» = `today − Срок устранения` (или
   `Completed − Срок устранения` для устранённых). Когда поле появится.
6. **C-6** Пересчитать «Просроченные» = строки, где Дней просрочки > 0
   и не устранено. «Критические» = просрочка > 30 дней.
7. **C-7** Карточки: добавить «Всего предписаний» и «Устранённые».
8. **C-8** Чекбокс «Не отображать устранённые предписания» (default = on).
9. **C-9** Фильтр «Период выданных предписаний» (по `CreationDate`).
10. **C-10** Фильтр «Блок» (после ответа на Q-27).
11. **C-11** Бар-чарт: оранжевые столбцы + синие пузырьки = всего
    неустранённых; внутри столбца = просроченные.
12. **C-12** Таблица: группировка + раскраска строк (розовый/салатовый).
13. **C-13** Сортировка: подрядчик ↑, просрочка ↓, критические вверху.
14. **C-14** Колонки таблицы: «Дата выдачи», «Статус предписания».
15. **C-15** UX: фиксировать ширину чарта, горизонтальная прокрутка.
