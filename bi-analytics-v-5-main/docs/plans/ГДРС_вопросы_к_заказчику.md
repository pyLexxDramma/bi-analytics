# Вопросы к заказчику по ГДРС (после сверки скринов «люди» / «техника»)

Дата: 2026-04-30  
Контекст: отчёт «График движения рабочей силы» / «ГДРС Техника», эталонная HTML-таблица план / СКУД / недели 1–6 / отклонение / дельта %.

---

## 1. Смысл колонки «СКУД» и связь с неделями 1–6

**Как сейчас в коде:** значение в колонке **«СКУД»** в строке считается как **среднее арифметическое** по шести столбцам «1 неделя» … «6 неделя» (сумма по неделям ÷ 6, с округлением).  
В строке **«Итого»** по неделям показываются **суммы** по всем строкам таблицы, а **«СКУД»** в «Итого» — это **среднее** этих шести сумм (то же правило).

**Вопрос:** так и задумано по макету Excel / ТЗ, или колонка **«СКУД»** должна быть:

- суммой значений по неделям за месяц,  
- или «снимком» факта из другого поля выгрузки,  
- или иное определение?

Нужно явное правило, чтобы таблица не воспринималась как ошибка («СКУД не равен сумме недель»).

---

## 2. Поведение при разных числах по неделям (техника)

На выгрузке **техника** по неделям часто разные числа; **люди** чаще показывают одинаковые значения во всех 6 колонках — тогда среднее совпадает с «узнаваемым» фактом.

**Вопрос:** для техники заказчик ожидает **именно среднюю неделю**, **сумму за месяц**, или колонку «СКУД» из файла без пересчёта из недель?

---

В PDF есть **противоречивые** фразы про красный/зелёный для отклонений и дельты.

**Вопрос:** зафиксировать единые правила (как в эталонном Excel: при каком знаке отклонения какой цвет для ячейки и для каких блоков).

В правках PDF фигурируют поля из **«Договор CSV»**.

В PDF упомянуто хранение **истории** помесячных файлов ресурсов для сравнения через ИИ.
