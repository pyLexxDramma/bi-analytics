## 2. Таблица. Состав столбцов (по строкам — подрядчик/договор)

| # | Столбец | Описание |
|---|---------|----------|
| 1 | **Договор стоимость** | Сумма по договору (стоимость) |
| 2 | **Всего выполненных обязательств по платежам** | Сумма всех произведённых оплат подрядчику (Аванс + КС-2) |
| 3 | **Аванс** | Сумма выданных авансов |
| 4 | **Выполнено (подписано КС-2)** | Сумма по подписанным КС-2 |
| 5 | **Остаток** | Стоимость договора − Всего выполненных обязательств по платежам |
| 6 | **Отклонение** | Стоимость договора − Всего выполненных обязательств по платежам (или другая формула — уточнить) |
| 7 | **КС-2 − Аванс** | Производная колонка: разница между суммой по подписанным КС-2 и суммой авансов |

### Цвет столбца «Отклонение»
- **<0 → красный** шрифт.
- **≥0 → зелёный** шрифт.

### Индикатор «Аванс ≥ 60% Договора»
- Если значение `Аванс / Договор стоимость ≥ 60 %` — **красный** индикатор напротив подрядчика.
- Иначе — **зелёный**.

## Источник данных и проблемы

### Текущая структура `web/1с_*_DK.json` / `_DK1.json` (по snapshot 2026-05-06)

В каждой строке **только следующие поля**:
- `Организация` (объект): `ID_Организации`, `НаименованиеОрганизации`.
- `Контрагент` (объект): `ID_Контрагента`, `НаименованиеКонтрагента`, `ИНН`, `КПП`.
- `Договор` (объект): `ID_Договора`, `НомерДоговора`, `ДатаДоговора`.
- `ОстатокНаНачало`, `ОстатокНаНачалоПериода`, `ОстатокНаНачалоПериодаПоАвансам`.
- `ВсегоОплат`, `ВсегоОплат_Аванс`.
- `ОстатокНаКонец`, `ОстатокНаКонецПериода`, `ОстатокНаКонецПериодаПоАвансам`.

### Чего НЕТ в источнике
1. **«Договор стоимость»** — суммы по договору в выгрузке нет (видим только остатки и оплаты).
2. **«КС-2» (подписано)** — отдельной величины нет; есть только `ВсегоОплат` и `ВсегоОплат_Аванс`. Возможно, КС-2 = `ВсегоОплат − ВсегоОплат_Аванс`, но это **догадка**.
3. **«Период» как ось X** — DK.json — это **snapshot на дату** (одна строка = баланс по договору на дату выгрузки). Чтобы построить «период по оси X», нужно **складывать несколько snapshot-ов** (накопить серию по датам файлов `1с_*_DK.json`). Поддерживается, но требует доработки.
4. **«Функциональный блок» / «Строение»** — в DK.json этих разрезов нет; они есть только в MSP-выгрузке (`msp_*.csv`). Нужен **маппинг через договор/подрядчика → MSP**.
