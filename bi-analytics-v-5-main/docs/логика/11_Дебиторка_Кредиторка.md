# Дебиторская и кредиторская задолженность подрядчиков

## Поля интерфейса ↔ FTP ↔ маппинг
Источник FTP: `web/*_DK.json`

| Поле в интерфейсе | Колонка FTP | Canonical/Admin key |
|---|---|---|
| Организация | `Организация` | `organization` |
| Контрагент | `Контрагент` | `contractor` |
| Договор | `Договор` | `contract` |
| Остаток на начало | `ОстатокНаНачалоПериода` | `opening_balance` |
| Остаток на начало (авансы) | `ОстатокНаНачалоПериодаПоАвансам` | `opening_advance_balance` |
| Оплаты | `ВсегоОплат` | `total_payments` |
| Оплаты авансом | `ВсегоОплат_Аванс` | `total_advance_payments` |
| Остаток на конец | `ОстатокНаКонецПериода` | `closing_balance` |
| Остаток на конец (авансы) | `ОстатокНаКонецПериодаПоАвансам` | `closing_advance_balance` |

## Чек-боксы
- `Показывать нетто (без авансов)`: переключает на расчет остатка за вычетом авансов.
- `Группировать по контрагенту`: меняет уровень агрегирования в таблице/графике.

## Фильтры
- **Организация**, **Контрагент**, **Договор**, **Период**.

## Логика расчета
- Нетто начало = `opening_balance - opening_advance_balance`.
- Нетто конец = `closing_balance - closing_advance_balance`.
- Динамика оплат: агрегирование `total_payments` по периоду/контрагенту.

## Вывод (что видит пользователь)
- **KPI**: ключевые остатки и оплаты.
- **Графики**: динамика задолженности/оплат.
- **Таблица**: детализация по договорам и контрагентам.

## Контролы UI (как в коде)

- `selectbox`: `Тип подрядчика`
- `selectbox`: `Договор`
- `selectbox`: `Проект`
