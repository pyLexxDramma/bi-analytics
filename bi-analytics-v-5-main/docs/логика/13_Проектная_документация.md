# Проектная документация

## Поля интерфейса ↔ FTP ↔ маппинг
Источники FTP:
- факт: `web/tessa_*_rd.csv`
- план: `web/AI/other_*_rd.csv`

| Поле в интерфейсе | Колонка FTP | Canonical/Admin key |
|---|---|---|
| Проект | `ObjectName` / `ObjectProjectName` | `project name` |
| Шифр ПД | `DivisionCipher` / `Шифр` | `division_cipher` |
| Раздел | `SubDivisionVersionName` / `Наименование раздела` | `section_name` |
| Плановая дата выдачи | `Дата выдачи разделов по Договору` | `pd_plan_date` |
| Фактическая дата | `CreationDate` | `pd_fact_date` |
| Статус | `KrState` | `pd_status` |

## Чек-боксы
- `Только просроченные`: показывает строки с положительным отклонением по дате.
- `Только выданные`: оставляет документы, у которых есть факт выдачи.

## Фильтры
- **Проект**, **Период**, **Раздел/шифр**, **Статус**.

## Логика расчета
- Связка плана и факта по проекту и разделу.
- Отклонение (дни) = `pd_fact_date - pd_plan_date`.
- Статусные KPI: в срок / просрочено / не выдано.

## Вывод (что видит пользователь)
- **Графики**: выполнение ПД по срокам.
- **Таблица**: детальные строки ПД (план, факт, отклонение, статус).

## Контролы UI (как в коде)
- `tabs`: `Проектная документация`
