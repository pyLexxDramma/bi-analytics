# ГДРС (техника)

## Поля интерфейса ↔ FTP ↔ маппинг
Источники FTP:
- факт: `web/AI/other_*_resursi.csv`
- план: `web/1c_*_spravochniki.json`

| Поле в интерфейсе | Колонка FTP | Canonical/Admin key |
|---|---|---|
| Проект | `Проект` / `Наименование_Проекта` | `project name` |
| Подрядчик | `Подрядчик` / `Наименование_Контрагента` | `contractor` |
| Тип техники | `тип ресурсов` (строки техники) | `equipment_type` |
| Факт техника | дневные/недельные колонки CSV | `fact equipment` |
| План техника | `КоличествоСпецТехники` | `gdrs_plan_equipment` |

## Чек-боксы
- `Только техника`: оставляет только тип ресурса техника.
- `Показывать доли`: добавляет долю подрядчика/типа техники.

## Фильтры
- **Проект**, **Подрядчик**, **Тип техники**, **Период**.

## Логика расчета
- Факт агрегируется по периоду и подрядчику.
- План мержится из 1C справочников по контрагенту/проекту.
- `Отклонение = Факт - План`.
- Цвет отклонения:
  - красный, если `Факт < План`,
  - зеленый, если `Факт >= План`.

## Вывод (что видит пользователь)
- **KPI**: факт, план, отклонение.
- **Графики**: план/факт, распределение по подрядчикам/типам.
- **Таблица**: детализация по периодам и контрагентам.

## Контролы UI (как в коде)
- В этой функции явные `st.*` контролы локально не объявлены (используются общие фильтры/обертки).
