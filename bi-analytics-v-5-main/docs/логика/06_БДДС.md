# БДДС

## Поля интерфейса ↔ FTP ↔ маппинг
Источники FTP:
- `web/AI/msp_*_<date>.csv` (иерархия/лот/этап),
- `web/*_dannye.json` (финансовые суммы БДДС).

| Поле в интерфейсе | Колонка FTP | Canonical/Admin key |
|---|---|---|
| Лот | `ЛОТ` (MSP) | `lot` |
| Этап | задача уровня 2 (MSP `Название`, `Уровень`) | `stage_l2` |
| Период | `Период` | `period` |
| Сценарий | `Сценарий` | `scenario` |
| Проект | `Проект` | `project name` |
| Статья | `СтатьяОборотов` | `turnover item` |
| Сумма | `Сумма` | `amount` |
| Доход/Расход | `РасходДоход` | `income_expense_flag` |
| Тип статьи | `ТипСтатьи` | `item_type` |
| План БДДС | `Сценарий` содержит `ПЛАН`, `СтатьяОборотов` не содержит `БДР` | `bdds_plan` |
| Факт БДДС | `Сценарий` содержит `ФАКТ`, `СтатьяОборотов` не содержит `БДР` | `bdds_fact` |
| Отклонение | вычисляемое | `bdds_fact - bdds_plan` |

## Чек-боксы
- `Накопительно`: переключает формулу с помесячной суммы на cumulative.
- Дополнительные чек-боксы (если включены) управляют только отображением статей.

## Фильтры
- **Проект**, **Период**, **Сценарий**, **Статья/тип статьи**.
- **Лот**, **Этап** (по MSP-иерархии: этап = задачи уровня 2).

## Логика расчета
- Для проекта берутся дочерние задачи MSP, где заполнен `ЛОТ`.
- Лот определяется по дочерним задачам в ветке родителя (идентификатор дочерней > родительской и < следующего родителя).
- Этап = название задачи уровня 2 в MSP.
- План БДДС: `Сценарий` содержит `ПЛАН`, `СтатьяОборотов` без `БДР`, далее `sum(amount)` по выбранным срезам.
- Факт БДДС: `Сценарий` содержит `ФАКТ`, `СтатьяОборотов` без `БДР`, далее `sum(amount)` по выбранным срезам.
- Отклонение: `БДДС факт - БДДС план`.
- Если данных по части колонок нет, выводится `Н/Д`.
- Для накопительного режима: `cumsum` по периоду в хронологическом порядке.

## Вывод (что видит пользователь)
- **Графики**: bar/line по месяцам или накопительно.
- **Таблица**: свод по проекту, статьям и суммам с экспортом.
- Финансовые суммы показываются в **млн рублей**, формат до **2 знаков** после запятой.
- Цвет отклонения: `> 0` красный, `< 0` зеленый.
