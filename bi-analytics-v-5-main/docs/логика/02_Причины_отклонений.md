# Причины отклонений

## Поля интерфейса ↔ FTP ↔ маппинг
Источник FTP: `web/AI/msp_*_<date>.csv`

| Поле в интерфейсе | Колонка FTP | Canonical/Admin key |
|---|---|---|
| Причина отклонения | `Причины_отклонений` | `reason of deviation` |
| Проект | `ID_проекта` / `Проект` | `project id` / `project name` |
| Блок | `БЛОК` | `block` |
| Лот | `ЛОТ` | `lot` |
| План окончание | `Окончание` | `plan end` |
| База окончание | `Базовое_окончание` | `base end` |
| Отклонение окончания | `Отклонение_окончания` | `deviation in days` |

## Вкладки
- `Доли причин по проекту`
- `Динамика отклонений по месяцам`
- `Динамика причин`

## Чек-боксы
- Если чек-боксы доступны в текущем экране, они используются как режимы отображения (например, скрывать пустые причины).
- Формулы расчета долей не меняются, меняется только набор строк.

## Фильтры
- **Проект**: оставляет строки выбранного проекта.
- **Блок/лот/раздел**: сужает причину в нужной части проекта.
- **Период**: фильтрация по датам задач (месяц/интервал).
- **Причина**: локальный фильтр таблицы детализации.

## Логика расчета
- Берутся строки MSP, где есть причина или есть отклонение срока.
- Доля причины = `count(строк причины) / count(всех строк с причинами)` в текущем фильтре.
- Для динамики: `groupby(период, причина)` с подсчетом количества.

## Вывод (что видит пользователь)
- **Диаграмма**: круговая (доли причин).
- **Графики**: столбчатые/фасетные по периодам и причинам.
- **Таблица**: детальные строки задач с причиной и сроками, с выгрузкой.

## Контролы UI (как в коде)
- В этой функции явные `st.*` контролы локально не объявлены (используются общие фильтры/обертки).
