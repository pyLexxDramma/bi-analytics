# Девелоперские проекты

## Поля интерфейса ↔ FTP ↔ маппинг
Источник FTP: `web/AI/msp_*_<date>.csv`

| Поле в интерфейсе | Колонка FTP | Canonical/Admin key |
|---|---|---|
| Проект | `ID_проекта` / `Проект` | `project id` / `project name` |
| Блок | `БЛОК` | `block` |
| Лот | `ЛОТ` | `lot` |
| Раздел | иерархия MSP | `section` |
| Задача | `Название` / `Название задачи` | `task name` |
| План начало | `Начало` | `plan start` |
| План окончание | `Окончание` | `plan end` |
| База начало | `Базовое_начало` | `base start` |
| База окончание | `Базовое_окончание` | `base end` |
| % выполнения | `Процент_завершения` | `pct complete` |

## Чек-боксы
- Явных чек-боксов обычно нет.
- Если в конкретной сборке включены служебные чек-боксы, они работают только как ограничение выборки (не меняют формулы).

## Фильтры
- **Проект**: ограничивает все графики/таблицы выбранным проектом.
- **Блок/лот/раздел**: сужают задачи до организационного среза.
- **Период**: оставляет строки, где дата задачи попадает в диапазон.

## Логика расчета
- Колонки MSP маппятся в canonical-ключи.
- Для повторных снимков используется последний `snapshot_date` по проекту.
- Считаются сводные показатели по отфильтрованным задачам: количество, средний `% выполнения`, отклонения по срокам.

## Вывод (что видит пользователь)
- **Таблицы**: детализация задач с датами и статусами.
- **Графики**: агрегаты по проекту/блоку (bar/line в зависимости от среза).
- **Диаграммы**: распределения по статусам/этапам (если включены в текущую конфигурацию экрана).

## Требования к отображению (ТЗ)

1. **Выводится только одна дата на задачу и на колонку**  
   - В **матрице контрольных точек** в каждой ячейке «План» / «Факт» — одна дата в формате `ДД.ММ.ГГГГ` (или «Н/Д»); в «Откл.» — одно текстовое значение отклонения (например `0 дн.`, `+N дн.`).  
   - В **таблице задач** в колонках дат — по одному значению `дд.мм.гггг` на ячейку (`dashboard_developer_projects`, форматирование `_fmt_date`).

2. **Выводится только обозначенный список задач для вывода**  
   - Строки матрицы задаются фиксированным перечнем вех/блоков в коде + при необходимости расширяются настройкой вех (JSON в админке, тот же контур, что и «Контрольные точки»): `build_dev_tz_matrix_rows` / `get_control_point_milestones_effective` в `dashboards/dev_projects_tz_matrix.py`.  
   - Таблица задач ниже — только строки MSP с хотя бы одной парой дат (план или факт), см. отбор `_has_base | _has_fact` в `dashboard_developer_projects`.

3. **Подсвечиваются строки со значениями по заданной логике**  
   - **Матрица**: при срабатывании флага предупреждения по вехе подсвечиваются ячейки **План / Факт / Откл.** этой строки (класс `dev-tz-warn` в CSS матрицы).  
   - **Таблица задач**: числовые отклонения окрашиваются — отклонение начала `< 0` и отклонение окончания `> 0` красным, иначе зелёным (`_dev_start_color` / `_dev_end_color`).

4. **Если у задачи в MSP столбце «% выполнения» не равен 100%, тогда значение ячейки подсветить рыжим**  
   - Логика: `_is_pct_complete_not_100` в `dev_projects_tz_matrix.py` (учёт доли `0…1` и процентов `0…100`).  
   - В матрице девелоперских проектов оранжевая подсветка ячеек строки вехи при `warn` (связано с `%` по matched задачам вехи).  
   - В отдельной таблице «Контрольные точки» для вех **ГПЗУ** и **Экспертиза стадии П** (`slug` `gpzu`, `exp_pd`) оранжевая подсветка ячейки при `warn_pct` — см. `CONTROL_POINTS_ORANGE_PCT_SLUGS` и класс `cp-td-warn`.

5. **Нет дублирования проектов и задач**  
   - Дубликаты логических проектов в фильтре сводятся через `_unique_project_labels_for_select` / канонизацию `project name` (`_project_column_apply_canonical`).  
   - При дублирующихся одноимённых колонках MSP в одной строке берётся первое непустое значение (`_series_first_value`).  
   - Для ячейки вехи при нескольких matched задачах берётся одна представительная строка (`sort_values` по имени задачи, `iloc[0]` в `_one_milestone_cell`).

6. **При фильтрации по проекту в таблице для экспорта и таблице с контрольными точками выводятся данные только по выбранному проекту**  
   - `st.selectbox("Проект", …)` → при выборе не «Все» применяется фильтр `filtered` по `_project_filter_norm_key` (`dashboard_developer_projects`).  
   - Экспорт матрицы (`developer_projects_matrix_*.csv`) и таблица задач строятся из того же `filtered` / блоков, собранных после фильтра.

7. **Числовые значения выводятся в целочисленном формате, без знаков после запятой**  
   - В таблице задач колонки «Отклонение начала», «Отклонение окончания», «Базовая длительность», «Длительность» форматируются через `_fmt_int` (округление до целого, вывод без дробной части).

8. **Финансовые показатели выводятся в млн рублей, до двух знаков после запятой**  
   - Строка «Выборка ДС, млн руб.» в матрице: формат `_fmtml` — **два** знака после запятой, запятая как десятичный разделитель (`dev_projects_tz_matrix.py`).

## Контролы UI (как в коде)
- В этой функции явные `st.*` контролы локально не объявлены (используются общие фильтры/обертки).
