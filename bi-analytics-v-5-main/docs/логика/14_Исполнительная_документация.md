# Исполнительная документация

## Поля интерфейса ↔ FTP ↔ маппинг
Источники FTP:
- `web/tessa_*_id.csv`
- `web/tessa_*_task.csv`
- `web/KrStates.csv`

| Поле в интерфейсе | Колонка FTP | Canonical/Admin key |
|---|---|---|
| ID карточки | `DocID` / `CardID` | `doc_id` / `card_id` |
| Вид документа | `KindName` | `kind_name` |
| Статус KR | `KrState`, `KrStateID` | `kr_state`, `kr_state_id` |
| Дата создания | `CreationDate` | `created_at` |
| Проект/объект | `ObjectName` | `object_name` |
| Лот | `Lot` | `lot` |
| Результат задачи | `Result` | `task_result` |
| Дата завершения задачи | `Completed` | `completed_at` |

## Чек-боксы
- `Показывать только незавершенные`: фильтрует по статусам/отсутствию `completed_at`.
- `Группировать по статусу`: меняет представление таблицы/графика на агрегированное.

## Фильтры
- **Проект/объект**, **Период**, **Статус**, **Лот**, **Тип документа**.

## Логика расчета
- Джойн `DocID` (ID) с `CardID` (TASK).
- Статус расшифровывается через `KrStates`.
- KPI считаются по финальному состоянию карточки и задач.

## Вывод (что видит пользователь)
- **KPI**: количество документов по статусам.
- **Графики**: распределение и динамика по статусам.
- **Таблица**: детальная карточка документа + связанная задача.

## Контролы UI (как в коде)
- `selectbox`: `Объект`
- `selectbox`: `Контрагент`
- `selectbox`: `Вид документа`
- `tabs`: `Накопительным итогом`
