# Контрольные точки

## Поля интерфейса ↔ FTP ↔ маппинг
Источник FTP: `web/AI/msp_*_<date>.csv`

| Поле в интерфейсе | Колонка FTP | Canonical/Admin key |
|---|---|---|
| Веха/контрольная задача | `Название` | `task name` |
| Проект | `ID_проекта` | `project id` |
| План | `Базовое_окончание` | `base end` |
| Факт | `Окончание` | `plan end` |
| Откл. | вычисляемое | `base end - plan end` |
| Факт/статус | `Процент_завершения` | `pct complete` |
| Уровень | `Уровень` / `Уровень_структуры` | `level` / `level structure` |

## Правила маппинга (по ТЗ)
- `План` = `Базовое окончание` из CSV MSP.
- `Факт` = `Окончание` из CSV MSP.
- `Откл.` = `План - Факт` (в днях).
- Если по нужной колонке данных нет, в ячейке показывается `Н/Д`.

## Фильтры
- **Проект** (задачи ур.1).
- **Функциональный блок** (задачи ур.2).
- **Строения** (задачи ур.3).
- **Верхний уровень задач** (задачи ур.4).
- **Детальный уровень задач** (задачи ур.5).

## Логика расчета
- Берутся задачи проекта, парсится их иерархия (уровни/родители).
- Далее задачи сопоставляются с выбранным списком контрольных задач по правилам маппинга.
- Для каждой контрольной задачи выводится `План`, `Факт`, `Откл.`.
- Если данных нет, выводится `Н/Д` без падения визуализации.

## Вывод (что видит пользователь)
- **KPI карточки** по состоянию контрольных точек.
- **График** по статусам/срокам.
- **Таблица** с точками и датами (план/база/факт состояние).

## Контролы UI (как в коде)
- В этой функции явные `st.*` контролы локально не объявлены (используются общие фильтры/обертки).
