# Контрольные точки

## Поля интерфейса ↔ FTP ↔ маппинг
Источник FTP: `web/AI/msp_*_<date>.csv`

| Поле в интерфейсе | Колонка FTP | Canonical/Admin key |
|---|---|---|
| Веха/контрольная задача | `Название` | `task name` |
| Проект | `ID_проекта` | `project id` |
| План дата | `Окончание` / `Базовое_окончание` | `plan end` / `base end` |
| Факт/статус | `Процент_завершения` | `pct complete` |
| Уровень | `Уровень` / `Уровень_структуры` | `level` / `level structure` |

## Чек-боксы
- Если включен чек-бокс “только контрольные задачи”, берутся только задачи, попадающие под правило в админке (`MSP: задача для метрик`).
- Если включен “скрыть завершенные”, исключаются задачи с `pct complete = 100`.

## Фильтры
- **Проект**: срез по проекту.
- **Период**: ограничение по датам контрольных точек.
- **Уровень/тип задачи**: оставляет только нужный слой иерархии.

## Логика расчета
- Выбор вех строится по правилам админки.
- Для каждой точки считается статус: в срок / с отклонением / завершено.
- KPI = количество точек по статусам и их доля.

## Вывод (что видит пользователь)
- **KPI карточки** по состоянию контрольных точек.
- **График** по статусам/срокам.
- **Таблица** с точками и датами (план/база/факт состояние).

## Контролы UI (как в коде)
- В этой функции явные `st.*` контролы локально не объявлены (используются общие фильтры/обертки).
