# Рабочая документация

## Поля интерфейса ↔ FTP ↔ маппинг
Источники FTP:
- факт: `web/tessa_*_rd.csv`
- план: `web/AI/other_*_rd.csv`
- статусы задач: `web/tessa_*_task.csv`

| Поле в интерфейсе | Колонка FTP | Canonical/Admin key |
|---|---|---|
| Проект | `ObjectName` / `ObjectProjectName` | `project name` |
| Шифр | `DivisionCipher` / `Шифр` | `division_cipher` |
| Раздел | `SubDivisionVersionName` / `Наименование раздела` | `section_name` |
| Плановая дата | `Дата выдачи разделов по Договору` | `rd_plan_date` |
| Фактическая дата | `CreationDate` | `rd_fact_date` |
| Статус | `KrState`, `Result`, `Completed` | `rd_status` |

## Чек-боксы
- `Только просроченные`: оставляет строки с `rd_fact_date > rd_plan_date`.
- `Скрыть без плана`: убирает строки, где нет `rd_plan_date`.

## Фильтры
- **Проект**, **Период**, **Раздел/шифр**, **Статус**.

## Логика расчета
- План/факт связываются по `проект + шифр + раздел`.
- Просрочка в днях = `rd_fact_date - rd_plan_date`.
- Агрегации: количество в срок / с просрочкой / без факта.

## Вывод (что видит пользователь)
- **Графики**: план/факт выдачи и динамика просрочки.
- **Таблица**: детализация разделов РД с датами и отклонением.

## Контролы UI (как в коде)
- `tabs`: `Рабочая документация`
