# Отклонение от базового плана

## Поля интерфейса ↔ FTP ↔ маппинг
Источник FTP: `web/AI/msp_*_<date>.csv`

| Поле в интерфейсе | Колонка FTP | Canonical/Admin key |
|---|---|---|
| План начало | `Начало` | `plan start` |
| План окончание | `Окончание` | `plan end` |
| База начало | `Базовое_начало` | `base start` |
| База окончание | `Базовое_окончание` | `base end` |
| Отклонение начала | `Отклонение_начала` | `deviation start days` |
| Отклонение окончания | `Отклонение_окончания` | `deviation in days` |
| % завершения | `Процент_завершения` | `pct complete` |
| Причина | `Причины_отклонений` | `reason of deviation` |
| Примечание | `Заметки` | `notes` |
| Лот | `ЛОТ` | `lot` |

## Чек-боксы и что они делают
- `Скрыть завершенные (100%)`: исключает строки с `pct complete = 100`.
- `Только ковенанты`: оставляет задачи из ветки ковенантов.
- `Только диаграммы с отрицательным отклонением`: показывает группы, где условие отклонения выполняется.
- `Показывать отклонение длительности`: добавляет колонку длительности в таблицу.

## Фильтры
- **Проект**: срез по проекту.
- **Блок/лот/раздел**: срез по структуре.
- **Период**: по датам задач.
- **Режим подписи** (`по MSP` / `по лоту`): меняет подписи категорий на графике.

## Логика расчета
- `diff_start_days = base start - plan start`.
- `diff_end_days = base end - plan end`.
- Основной показатель строки: приоритет `diff_end_days`, fallback `diff_start_days`.
- Агрегация по группам для bar chart: сумма/среднее отклонения в группе (в зависимости от среза).

## Вывод (что видит пользователь)
- **График**: горизонтальный bar chart, сортировка по величине отклонения.
- **Таблица**: детально по задачам, с датами, отклонениями, причиной.
- **Доп.блоки**: ковенанты/ЗОС (если такие строки есть в данных).

## Контролы UI (как в коде)
- В этой функции явные `st.*` контролы локально не объявлены (используются общие фильтры/обертки).
