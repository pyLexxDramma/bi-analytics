# Прогнозный бюджет

## Поля интерфейса ↔ FTP ↔ маппинг
Источник FTP: `web/*_dannye.json`

| Поле в интерфейсе | Колонка FTP | Canonical/Admin key |
|---|---|---|
| Факт | `Сумма` (факт-сценарий) | `fact amount` |
| Прогноз | `Сумма` (прогноз-сценарий/расчет) | `forecast amount` |
| План/база | `Сумма` (базовый сценарий) | `plan amount` |
| Период | `Период` | `period` |
| Проект | `Проект` | `project name` |

## Чек-боксы
- `Накопительно`: переключает на cumulative.
- `Сравнивать с утвержденным`: база сравнения = approved.

## Фильтры
- **Проект**, **Период**, **Статья/группа**, **Сценарий**.

## Логика расчета
- Прогноз берется из прогнозного контура или считается из факта+плана по правилам отчета.
- Отклонения:
  - `forecast - plan`,
  - `forecast - approved` (если включен режим).

## Вывод (что видит пользователь)
- **Графики**: факт/план/прогноз по периодам.
- **Таблица**: численные значения и отклонения.

## Контролы UI (как в коде)
- `checkbox`: `Скрыть отклонение`
