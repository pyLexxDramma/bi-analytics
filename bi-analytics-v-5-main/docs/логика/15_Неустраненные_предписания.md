# Неустраненные предписания

## Поля интерфейса ↔ FTP ↔ маппинг
Источники FTP:
- `web/tessa_*_id.csv`
- `web/tessa_*_task.csv`
- `web/KrStates.csv`

| Поле в интерфейсе | Колонка FTP | Canonical/Admin key |
|---|---|---|
| Предписание (карточка) | `DocID` | `doc_id` |
| Статус предписания | `KrState`, `KrStateID` | `kr_state`, `kr_state_id` |
| Дата выдачи | `CreationDate` | `issue_date` |
| Объект/проект | `ObjectName` | `object_name` |
| Лот | `Lot` | `lot` |
| Результат устранения | `Result` | `task_result` |
| Дата завершения | `Completed` | `completed_at` |

## Чек-боксы
- `Только просроченные`: оставляет предписания со сроком/датой закрытия вне норматива.
- `Исключить служебные статусы`: убирает строки со служебными KR статусами.

## Фильтры
- **Период выдачи**, **Период устранения**, **Статус**, **Объект/лот**.

## Логика расчета
- Дедупликация карточек по `doc_id`.
- Джойн со статусами задач.
- Классификация: устранено / не устранено / просрочено / критично.
- KPI и графики строятся на очищенной выборке.

## Вывод (что видит пользователь)
- **KPI** по группам статусов предписаний.
- **Графики**: динамика и распределение (в т.ч. просроченные).
- **Таблица**: детальный перечень предписаний.
