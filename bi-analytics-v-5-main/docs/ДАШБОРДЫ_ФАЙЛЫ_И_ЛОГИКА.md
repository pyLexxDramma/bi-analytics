# Дашборды: файлы и логика

Цель документа: для каждого дашборда зафиксировать, из каких файлов берутся данные и как работает логика расчёта/визуализации.

## 1) Девелоперские проекты
- **Источник файлов:** `web/AI/msp_*.csv` (основной), частично справочники/служебные поля из `web/`.
- **Логика:** построение сводных метрик по проектам на основе MSP-структуры (уровни, блоки, сроки, проценты выполнения), с фильтрами по проектам/блокам/периодам.

## 2) Причины отклонений
- **Источник файлов:** `web/AI/msp_*.csv`.
- **Логика:** берутся строки задач MSP с заполненной причиной отклонения и/или флагом отклонения; строятся:
  - доли причин,
  - динамика по месяцам,
  - динамика причин.
  Фильтры: проект, функциональный блок, строение, период, причина.

## 3) Отклонение от базового плана
- **Источник файлов:** `web/AI/msp_*.csv`.
- **Логика:** расчёт отклонений по датам:
  - `Отклонение начала = base start - plan start`,
  - `Отклонение окончания = base end - plan end`.
  Строятся графики и таблицы по задачам/группам с сортировкой по величине отклонения.

## 4) Контрольные точки
- **Источник файлов:** `web/AI/msp_*.csv`.
- **Логика:** отбор контрольных задач (вех) по настройке в админке и правилам уровня MSP; агрегация по проектам/периодам, визуализация статуса исполнения точек.

## 5) График проекта
- **Источник файлов:** `web/AI/msp_*.csv`.
- **Логика:** Gantt/временные ряды по задачам с план/факт датами, фильтрами по проекту/уровню/структуре, подсветкой отклонений и причин.

## 6) БДДС
- **Источник файлов:** `web/*_dannye.json` (1C), при необходимости `web/*_spravochniki.json` для справочных связок.
- **Логика:** агрегирование денежных потоков по периодам и сценариям, группировка по статьям/проектам, отображение «месяцы» и «накопительно».

## 7) БДР
- **Источник файлов:** `web/*_dannye.json` (1C), частично `web/*_spravochniki.json`.
- **Логика:** аналогично БДДС, но для статей доходов/расходов БДР с агрегацией по периоду и проекту.

## 8) Бюджет план/факт
- **Источник файлов:** `web/AI/msp_*.csv`, fallback: `web/*_dannye.json` (1C).
- **Логика:** сравнение плановых и фактических бюджетов по проектам/лотам/периодам; при отсутствии корректного плана в MSP применяется fallback-логика из 1C.

## 9) Утверждённый бюджет
- **Источник файлов:** `web/*_dannye.json`, `web/*_spravochniki.json`.
- **Логика:** отображение утверждённых значений бюджета и сравнение с фактом по периодам/разрезам.

## 10) Прогнозный бюджет
- **Источник файлов:** `web/*_dannye.json`.
- **Логика:** расчёт прогнозных значений на основе факта и доступных плановых/сценарных данных с разрезами по проектам и периодам.

## 11) Дебиторская и кредиторская задолженность подрядчиков
- **Источник файлов:** `web/*_DK.json`.
- **Логика:** расчёт и показ остатков (в т.ч. нетто при наличии колонок авансов), оплат и динамики задолженности по подрядчикам/договорам/проектам.

## 12) Рабочая документация
- **Источник файлов:** `web/tessa_*_rd.csv` (факт), `web/AI/other_*_rd.csv` (план), доп. из `web/tessa_*_task.csv`.
- **Логика:** сопоставление план/факт по разделам РД (проект + шифр + раздел), расчёт просрочки и статусов выдачи.

## 13) Проектная документация
- **Источник файлов:** `web/tessa_*_rd.csv`, `web/AI/other_*_rd.csv`.
- **Логика:** аналогично РД, но в срезе ПД и соответствующих метрик/фильтров.

## 14) Исполнительная документация
- **Источник файлов:** `web/tessa_*_id.csv`, `web/tessa_*_task.csv`, `web/KrStates.csv`.
- **Логика:** объединение карточек ИД и задач согласования, расчёт статусов/сроков/просрочек, фильтрация служебных статусов.

## 15) Неустранённые предписания
- **Источник файлов:** `web/tessa_*_id.csv`, `web/tessa_*_task.csv`, `web/KrStates.csv`.
- **Логика:** дедуп по DocID/CardID, расчёт KPI (всего/устранено/неустранено/просрочено/критично), фильтры по датам выдачи и сроку устранения.

## 16) ГДРС (рабочая сила)
- **Источник файлов (факт):** `web/AI/other_*_resursi.csv`.
- **Источник файлов (план):** `web/1c_*_spravochniki.json` (`КоличествоРаботников`).
- **Логика:** факт берётся по СКУД/ресурсам, план подставляется через сопоставление контрагент+проект; расчёт `Отклонение = Факт - План`, показ по проектам/подрядчикам/неделям.

## 17) ГДРС Техника
- **Источник файлов (факт):** `web/AI/other_*_resursi.csv` (строки/колонки техники).
- **Источник файлов (план):** `web/1c_*_spravochniki.json` (`КоличествоСпецТехники`).
- **Логика:** аналогично ГДРС-рабочим, но по технике (отдельный срез `data_source`/`тип ресурсов`).

## 18) Просрочка выдачи РД / ПД
- **Источник файлов:** `web/tessa_*_rd.csv`, `web/AI/other_*_rd.csv`.
- **Логика:** расчёт разницы плановой и фактической даты выдачи, агрегации по проектам/разделам, таблицы и графики просрочки.
