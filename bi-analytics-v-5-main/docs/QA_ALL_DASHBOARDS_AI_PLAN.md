## Цикл проверки одного пункта (для исполнителя/ИИ)

Для **каждого** дашборда ниже проходить шаги по порядку. Пока шаг не закрыт, финальный чекбокс не ставить.

- **A. Запрос:** запросить у заказчика **скриншот** нужной таблицы/графика + **краткую формулировку ТЗ**.
- **B. Вкладка:** открыть в приложении указанный отчёт/вкладку и проверить реализацию.
- **C. Исправление:** при расхождении с ТЗ — правка в коде/данных.
- **D. Повторный скрин:** снова запросить скрин после правки.
- **E. Сверка с эталоном `web/`:** CSV/JSON по проекту и метрике.
- **F. Корректировка:** при несоответствии эталону — доработка и снова D–E.
- **G. Подтверждение заказчика:** явное **«ок»** в переписке.
- **H. Релиз:** `git commit` + `git push` на **main** + merge в **release**.
- **I. Чекбокс пункта** в этом файле и в [`DEV_PROJECTS_TZ_MATRIX_CHECKLIST.md`](DEV_PROJECTS_TZ_MATRIX_CHECKLIST.md) — отмечен.
