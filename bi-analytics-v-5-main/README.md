# BI Analytics Dashboard

Веб-приложение для бизнес-аналитики и визуализации данных проектов, построенное на Streamlit. Система предоставляет инструменты для анализа план-факт данных, управления пользователями и настройки отчетов.

## 🚀 Основные возможности

### Визуализация данных
- **Анализ дат**: Сравнение плановых и фактических сроков выполнения задач
- **Анализ бюджета**: Сравнение планового и фактического бюджета проектов
- **Интерактивные графики**: Gantt-диаграммы, scatter plots, bar charts
- **Фильтрация данных**: По проектам, секциям, статусу отклонений

### Система авторизации
- Многоуровневая система ролей (Суперадминистратор, Администратор, Менеджер, Аналитик)
- Безопасное хранение паролей (SHA-256)
- Восстановление пароля через токены
- Управление пользователями

### Административная панель
- Управление пользователями и ролями
- Настройка путей к файлам данных
- Просмотр логов действий пользователей
- Управление правами доступа к проектам
- Настройка фильтров по умолчанию
- Принудительное обновление отчетов

### Параметры отчетов
- Редактирование параметров отчетов аналитиками
- Настройка задач для расчета план-факта
- Корректировки плана бюджета и графика проекта
- История изменений параметров

## 📋 Требования

## 🔧 Установка

### 1. Клонирование репозитория

```bash
git clone https://github.com/mk-im/bi-analytics.git
cd bi-analytics
```

### 2. Создание виртуального окружения

```bash
# Windows
python -m venv venv
venv\Scripts\activate

# Linux/Mac
python3 -m venv venv
source venv/bin/activate
```

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

## 🏃 Запуск приложения

### Вариант 1: Использование bat-файла (Windows)

```bash
start_visualization_app.bat
```

### Вариант 2: Ручной запуск

```bash
# Активируйте виртуальное окружение
venv\Scripts\activate  # Windows
# или
source venv/bin/activate  # Linux/Mac

# Запустите приложение
streamlit run project_visualization_app.py
```

### Вариант 3: Прямой запуск без активации venv (Windows)

```bash
venv\Scripts\streamlit.exe run project_visualization_app.py
```

После запуска приложение будет доступно по адресу: **http://localhost:8501**

## 👤 Первый вход в систему

При первом запуске создаётся база данных `users.db`. Дефолтный суперадминистратор создаётся **только если** заданы переменные окружения:

- `DEFAULT_ADMIN_USERNAME` — логин первого администратора
- `DEFAULT_ADMIN_PASSWORD` — пароль (не храните в коде, только в `.env` или окружении развёртывания)

Скопируйте `.env.example` в `.env`, задайте значения и запустите приложение. После первого входа рекомендуется сменить пароль или удалить переменные из окружения.

⚠️ **Важно:** В репозитории нет ключей и паролей — их задаёт получатель проекта при развёртывании.

## 📊 Формат данных

Приложение работает с CSV и Excel файлами. Требуемые колонки:

| Колонка | Тип | Описание |
|---------|-----|----------|
| project name | string | Название проекта |
| abbreviation | string | Аббревиатура проекта |
| section | string | Секция/категория |
| task name | string | Название задачи |
| base start | date | Фактическая дата начала |
| base end | date | Фактическая дата окончания |
| plan start | date | Плановая дата начала |
| plan end | date | Плановая дата окончания |
| deviation | boolean | Наличие отклонения |
| deviation in days | integer | Количество дней отклонения |
| reason of deviation | string | Причина отклонения |
| budget plan | integer | Плановый бюджет |
| budget fact | integer | Фактический бюджет |

## 🐛 Устранение неполадок

### Проблема: "No valid date data available"
**Решение:** Проверьте формат дат в файле данных. Поддерживаются различные форматы дат.

### Проблема: "No valid budget data available"
**Решение:** Убедитесь, что колонки бюджета содержат числовые значения.

### Проблема: Ошибка авторизации
**Решение:** Проверьте, что база данных `users.db` создана и доступна для записи.

### Проблема: Порт занят
**Решение:** Streamlit автоматически попробует следующий доступный порт, или укажите другой порт:
```bash
streamlit run project_visualization_app.py --server.port 8502
```
