# Эталон выгрузок в `web/` (зафиксировано по файлам в репозитории)

Использовать при маппинге и доработке кода. Обновлять при смене формата 1С/выгрузок.

## `1с_*_spravochniki.json` (пример: `1с_27-03-2026_07-30_spravochniki.json`)

- Формат: JSON-массив объектов, **51** строка в образце.
- Поля (единственные в образце):
  - `ID_Контрагента` (GUID)
  - `Наименование_Контрагента`
  - `ИНН_Контрагента`
  - `КПП_Контрагента`
- **Плановых полей ГДРС (чел.-дни и т.п.) в этой поставке нет** — это справочник контрагентов. Связь с фактом ресурсов — по нормализованному наименованию (и при появлении в выгрузках — по `ID_Контрагента`).

## `1с_*_dannye.json` (пример: `1с_27-03-2026_07-30_dannye.json`)

- Обороты/движения: `Период`, `Сценарий`, `Контрагент`, `Проект`, `Сумма`, `РасходДоход`, `ТипСтатьи`, `ДоговорКонтрагента` и др. (см. первый элемент массива).
- **Не** эквивалент плану ГДРС по СКУД/людям в чел.-днях.

## `web/AI/other_*_resursi.csv` (пример: `other_01-01-2026_resursi.csv`)

- Многоуровневый заголовок; разделитель `;`.
- Смысловые колонки: `Проект`, `Подрядчик`, `тип ресурсов` (`рабочие` / `техника`), далее **суточные** колонки `ДД.ММ.ГГГГ` и/или `среднее значение за день` по блокам недель, в конце — `среднее значение количество ресурсов в день за месяц`.
- **Колонки «План» в шапке нет** — план-факт в дашборде ждёт либо колонку плана в отдельной выгрузке, либо расширение `spravochniki`/другого источника 1С с плановыми числами по контрагенту/проекту.

## Связь с кодом

- `load_all_from_web()` — `spravochniki` → `st.session_state["reference_contractors"]`.
- `_gdrs_merge_plan_from_1c_spravochniki()` в `dashboards/_renderers.py` подставляет план, **если** в `reference_contractors` появится числовая колонка с планом (эвристика по имени). В текущей поставке merge по плану не срабатывает — ожидаемо.
