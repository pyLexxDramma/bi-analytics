# Эталон выгрузок в `web/` (зафиксировано по файлам в репозитории)

Использовать при маппинге и доработке кода. Обновлять при смене формата 1С/выгрузок.

## `1с_*_spravochniki.json` (пример: `1с_27-03-2026_07-30_spravochniki.json`)

- Формат: JSON-массив объектов (~**51** строка в образце). Строки **разной длины**: первые записи — только реквизиты контрагента; далее — связка **подрядчик / договор / проект** и **плановые объёмы**.
- **Контрагент / договор / проект (ссылки):**  
  `ID_Контрагента`, `Наименование_Контрагента`, при необходимости `ID_Договора`, `Наименование_Договора`, `Дата_Начала_Договора`, `Дата_Окончания_Договора`, `ID_Проекта`, `Наименование_Проекта`, `ID_Организации`, `Наименование_Организации` и т.д.
- **Плановые за период (ГДРС) — в образце 27.03.2026:**  
  - `КоличествоРаботников` — **массив** объектов `[{ "Дата": "YYYY-MM-DD", "Количество": "<число>" }, ...]` — **по людям (рабочие)**.  
  - `КоличествоСпецТехники` — то же по **спецтехнике**.  
  В дашборде план в числе для сравнения с фактом: по массиву берётся **max(Количество)** (согласованный пик/план в выгруженных срезах); при смене бизнес-правила — согласовать (сумма / последняя дата / другое).
- `ДатаПолученияИД` — при необходимости в отчётах по согласованиям (не план-факт в чел./ед. в днях).

## `1с_*_dannye.json` (пример: `1с_27-03-2026_07-30_dannye.json`)

- Обороты/движения: `Период`, `Сценарий`, `Контрагент`, `Проект`, `Сумма`, `РасходДоход`, `ТипСтатьи`, `ДоговорКонтрагента` и др.
- **Не** дублирует план ГДРС по согласованным `КоличествоРаботников` / `КоличествоСпецТехники` (это **фин** слой, не СКУД-люди/техника в тех же единицах).

## `web/AI/other_*_resursi.csv` (пример: `other_01-01-2026_resursi.csv`)

- Многоуровневый заголовок; разделитель `;`.
- `Проект`, `Подрядчик`, `тип ресурсов` (`рабочие` / `техника`), далее **факт** по дням / средние; отдельной колонки «План» в шапке **нет** — план подставляется из `spravochniki` (см. выше) при нуле в `План_numeric` после чтения CSV.

## Связь с кодом

- `load_all_from_web()` — `spravochniki` → `st.session_state["reference_contractors"]` (DataFrame: ячейки `Количество*` могут быть **list**).
- `dashboards/_renderers.py` — `_gdrs_merge_plan_from_1c_spravochniki`: план рабочих/техники по строке факта; разбор JSON-массивов — `_gdrs_plan_metric_from_spravochnik_cell` (max по `Количество`).
